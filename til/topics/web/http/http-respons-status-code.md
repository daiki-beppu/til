# HTTP レスポンスステータスコードについて

## HTTP レスポンスステータスコードとは

特定のレスポンスが正常に動作したのかを教えてくれるもの
レスポンスは 5 種類に分類されている

- 情報レスポンス `100 - 199`
- 成功レスポンス `200 - 299`
- リダイレクトメッセージ `300 - 399`
- クライアントエラーレスポンス `400- 499`
- サーバーエラーレスポンス `500- 599`

## レスポンスステータスコードの代表例

### 成功レスポンス

- `200 OK` リクエストの成功
  成功の意味はメソッドによって違う
  - `GET` リソースの取得が正常に行われメッセージ本文で転送
  - `PUT or POST` 操作結果が正常に行われメッセージ本文で転送
- `201 Created` リクエストが成功し新しいリソースが作成
  一般的に`PUT`や`POST`を送信した後のレスポンス

### リダイレクトメッセージ

- `301 Moved Permanently`
  リソースの URL が変更されたのでレスポンスで新しい URL を与える

### クライアントエラーレスポンス

- `404 Not Found`
  サーバーがリクエストされたリソースを発見できないこと示す
  ブラウザでは URL が無効であることを意味する
  API の場合、通信先は有効であるものの
  リソース自体が存在しないこと意味することもある
